## CI/CD Pipeline (GitHub Actions)

У репозиторії налаштовано повний pipeline у файлі `.github/workflows/flutter_ci.yml`:

1.  Analyze: Автоматична перевірка коду за допомогою `flutter analyze`. Забезпечено 0 warnings та 0 errors.
2.  Format: Перевірка форматування всього коду за допомогою `dart format --set-exit-if-changed .`.
3.  Test: Запуск модульних та віджет-тестів з генерацією звіту про покриття (`flutter test --coverage`).
4.  Build: Автоматична збірка APK (тільки для гілки main) після успішного проходження попередніх етапів.

## Аналіз покриття (Coverage Report)

Відповідно до вимог, проект покритий тестами на рівні понад 60%. Це включає тестування моделей Product, CartItem та бізнес-логіки сервісу CartService. Весь код проходить автоматичну перевірку під час кожного push у репозиторій.

## Оптимізація продуктивності

За допомогою Flutter DevTools було виявлено та виправлено 2 критичні проблеми продуктивності:

1.  Heavy computations in build(): Важкі обчислення перенесено в окремий Isolate за допомогою `compute()`, що розвантажило UI-потік.
2.  Unnecessary widget rebuilds: Використано `Selector` та `Consumer` (Provider) замість повного перемальовування екранів через `watch`.
